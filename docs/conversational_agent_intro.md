# 💬 대화형 AI 설계 에이전트 (Conversational Design Agent)

## 한 줄 소개
연구 목표를 말로 설명하기만 하면, AI가 의도를 이해하고 MCP 도구들을 자율적으로 호출해 실제 프라이머/프로브 설계 파이프라인을 실행한 뒤, 결과가 기준에 미달할 경우 스스로 원인을 분석하고 전략을 바꿔 재시도까지 하는 상시 대기형 연구 어시스턴트입니다.

---

## 핵심 기능

**1. 자연어 한 마디로 설계 시작 — MCP 도구 자율 호출 기반**
"SARS-CoV-2 RdRp 유전자를 표적으로 하는 qPCR 프라이머를 설계해줘" 같은 문장 하나면 충분합니다. 에이전트는 PlayMCP 프로토콜(2025-03-26) 호환 MCP 서버에 등록된 9개의 실제 도구(전략 수립·파이프라인 실행·작업 상태 조회·결과 기반 Q&A·전략 템플릿 조회·과거 프로젝트 메모리 조회·what-if 비교·커버리지 상세 분석·체세포 변이 올리고 설계) 중 상황에 맞는 것을 스스로 선택·조합해 호출하며, 요청 하나로 의도 분석부터 실제 9단계 바이오인포매틱스 파이프라인 실행까지 자율적으로 이어갑니다.

**2. 결과 미달 시 자가 성찰(Reflection) 후 자동 재시도**
설계 결과의 커버리지가 전략별 기준치에 못 미치면, 에이전트가 곧바로 포기하지 않습니다. `ScientificReflectionAgent`가 어떤 지표가 왜 기준에 미달했는지 원인을 구조화해 분석("반성")하고, 그 결과를 바탕으로 다른 전략으로 전환해 최대 3회까지 Plan → Act → Evaluate → Reflect → Re-plan 사이클을 자동으로 반복합니다. 각 회차의 판정(통과/재시도/실패)과 반성 내용, 전략 전환 이유가 모두 대화창 타임라인에 그대로 기록되어 "왜 이 전략으로 바뀌었는지"까지 투명하게 확인할 수 있습니다.

**3. 실시간 진행 상황 중계**
설계는 MAFFT 정렬 → 엔트로피 분석 → Primer3 후보 생성 → 특이도 필터 → 변이 커버리지 → 열역학 점수 → AI 효율성 예측 → 최종 랭킹 → 리포트 생성까지 9단계 실제 파이프라인으로 진행되며, 각 단계와 소요 시간이 대화창에 실시간으로 표시됩니다.

**4. 완료 후 즉석 결과 Q&A**
설계가 끝나면 후보 목록(순위·점수·서열·산물 크기)이 채팅창에 바로 카드로 나타나고, "1등 후보 커버리지 점수가 몇 점이야?" 같은 질문에 실제 계산된 값만 근거로 답합니다 — 존재하지 않는 수치를 지어내지 않는 것이 원칙입니다(Anti-Hallucination).

**5. 리포트 다운로드**
결과 카드에서 버튼 한 번으로 HTML 분석 리포트를 파일로 바로 저장할 수 있습니다.

---

## 실제로 나오는 결과
- 순위가 매겨진 프라이머/프로브 후보 목록 (커버리지·특이도·열역학·AI 효율성 점수를 가중 합산한 최종 점수 기준)
- 각 후보의 정방향/역방향 프라이머 서열, TaqMan 프로브 서열, 예상 증폭 산물 크기
- hg38 전체 게놈 기준 BLAST 오프타겟 특이도 검증 결과
- 반복이 있었다면 회차별 판정과 성찰 로그, 전략 전환 이력
- 다운로드 가능한 HTML 분석 리포트

모든 수치는 결정론적 Python 엔진(Primer3, BLAST 등)이 계산한 실제 값이며, AI는 도구 호출·전략 선택·성찰·설명에만 관여합니다.

---

## 연구원에게 어떤 도움이 되는가

- **진입 장벽 제거** — Primer3 파라미터나 MCP 도구 이름을 몰라도, 자연어로 목표만 말하면 에이전트가 적절한 도구를 스스로 골라 호출합니다.
- **실패해도 사람이 다시 설정할 필요 없음** — 첫 시도가 기준 미달이면 에이전트가 원인을 분석해 다른 전략으로 자동 전환·재시도하므로, "왜 안 됐지, 뭘 바꿔야 하지"를 연구자가 직접 판단할 필요가 줄어듭니다.
- **대기 시간의 불확실성 해소** — 9단계 각각의 진행률과, 반복이 있을 경우 몇 회차인지까지 실시간으로 보입니다.
- **결과 해석 비용 감소** — 최종 결과뿐 아니라 "왜 이 전략으로 바뀌었는지"까지 성찰 로그로 설명되어 결과 리뷰 시간이 단축됩니다.
- **신뢰할 수 있는 답변** — 모든 점수·판정은 결정론적 엔진 계산값에 근거하므로, 보고서에 인용 가능한 수준의 재현성을 갖습니다.
